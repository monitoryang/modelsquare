## 产品需求文档（PRD）：实时交互式模型广场平台

**文档版本：** 1.0  
**最后更新日期：** 2026年1月23日  
**作者：** [杨汶]

### 1. 产品概述与目标用户

#### 1.1 产品概述

"模型广场"是一个面向AI开发者、研究人员和企业用户的在线平台，支持模型发现、对比、测试与实时推理可视化。区别于现有仅支持静态文件上传的模型平台，本产品核心创新在于引入视频推流能力与低延迟结果渲染，实现对AI模型输出的实时交互式体验，适用于直播分析、远程调试、教学演示、边缘部署验证等场景。

#### 1.2 目标用户

| 用户类型 | 需求场景 |
| --- | --- |
| AI开发者 | 快速测试新模型在真实流数据下的表现；对比不同模型精度与延迟 |
| 算法研究员 | 可视化验证模型在动态场景中的鲁棒性；分享研究成果 |
| 企业技术团队 | 评估第三方模型是否满足业务实时性要求；集成API到自有系统 |
| 教育/培训人员 | 演示模型工作原理；开展互动式AI教学 |

### 2. 详细功能列表

#### 2.1 模型管理

- 模型上传（支持 ONNX、TensorRT、PyTorch格式）
- 模型元数据编辑（名称、描述、任务类型、输入/输出规范、性能指标）
- 模型版本管理
- 公开/私有模型权限控制

#### 2.2 模型搜索与发现

- 关键词搜索（模型名、标签、作者）
- 多维筛选（任务类型：目标检测、语义分割、多模态模型(需要输入多个模态)；框架；精度；推理速度（在不同显卡上的推理速度展示）））
- 排序（按热度、评分、更新时间）
- 收藏与关注功能

#### 2.3 模型测试

- 静态测试：上传图片或视频文件，运行推理并展示结果
- 实时测试：
  - 调用本地摄像头
  - 输入 RTMP/WebRTC 视频流地址
  - 实时渲染模型输出（如检测框、关键点、热力图等）
  - 测试历史记录与回放
- 自动记录并对比关键指标（FPS、延迟、显存占用、准确率）

#### 2.4 多种模型类型支持
- 支持分类模型，检测模型，分割模型，多模态模型，NLP模型

#### 2.5 视频推流与实时渲染

- 支持 RTMP、HLS、WebRTC 输入协议
- 自动适配主流分辨率与帧率（最高支持 1080p@30fps）
- 前端基于 WebGL / Canvas 实现低延迟叠加渲染（<500ms 端到端延迟）
- 支持暂停、截图、录制推理结果视频

#### 2.6 API 接口服务

RESTful API 支持：

1. (POST /api/v1/models/{model_id}/infer/image)
- 功能：上传单张图片，使用指定模型进行推理
- 输入：multipart/form-data，字段 image（支持 JPG/PNG）
- 输出：JSON 结构化结果（如检测框坐标、类别、置信度） + 可选渲染图 URL
2. (POST /api/v1/models/{model_id}/infer/video)：上传短视频（≤30s）进行批量推理
- 功能：上传短视频（≤30秒，MP4/H.264），逐帧推理
- 输入：multipart/form-data，字段 video
- 输出：每帧结果数组 + 可选合成视频 URL
3. (POST /api/v1/models/{model_id}/stream/start)：创建推流会话，返回推流地址
- 功能：上传短视频（≤30秒，MP4/H.264），逐帧推理
- 输入：multipart/form-data，字段 video
- 输出：每帧结果数组 + 可选合成视频 URL
4. (GET /api/v1/models)：查询流状态与结果
- 功能：列出当前用户可访问的模型（含公开模型）
- 支持查询参数：task_type, framework, keyword 等
- 输出：模型列表（含 model_id, 名称, 描述, 输入规范等）
5. (GET /api/v1/models/{model_id})：查询流状态与结果
- 功能：获取指定模型的详细信息与输入/输出规范（用于客户端预校验）
返回结构化 JSON 结果 + 可选渲染图像 URL

API Key 认证与调用频率限制

#### 2.7 用户与权限

- 用户注册/登录（支持邮箱 + OAuth）
- 设立超级用户和普通用户，普通用户不能上传模型，只能使用模型，超级用户可以进行模型管理（模型的上传+加载+卸载+删除）
- 超级用户有个人模型库进行管理，普通用户只能用公开模型库
- 团队协作（可选，V2）

### 3. 功能优先级（MVP 与后续版本）

#### MVP（最小可行产品，V1.0）

**目标：** 验证核心价值——实时推流 + 模型测试 + 基础搜索

- [x] 模型管理（模型的上传+加载+卸载+删除）与基础元数据管理
- [x] 模型关键词搜索 + 任务类型筛选
- [x] 单模型测试（支持图片上传）
- [x] 视频推流接入（RTMP） + 实时结果渲染（Canvas）
- [x] 基础 REST API（图片/视频推理）
- [x] 用户注册与模型私有化
- [x] 支持多模态模型输入即同时输入图片，文字，影评等输入给模型
- [x] 支持分割模型，目标检测模型
- [x] 支持多模型横向对比

#### V1.1（增强体验）

- 模型对比功能（2模型并行）
- WebRTC 推流支持
- 测试历史记录
- API 调用统计与限流

#### V2.0（平台化扩展）

- 团队协作与模型共享
- 模型评分与评论系统
- 自动性能基准测试（在标准数据集上跑分）
- 模型微调/部署一体化（可选）

### 4. 界面设计要求

#### 4.1 设计原则

- 简洁高效：减少操作步骤，突出核心功能（测试/对比）
- 实时反馈：推流画面与模型输出同屏展示，延迟可视化
- 响应式布局：适配桌面端（主）与平板（次）

#### 4.2 核心页面

- 首页：搜索栏 + 热门模型推荐 + 快速测试入口
- 模型详情页：元信息 + 在线测试区（含推流开关）+ 下载/API 按钮
- 个人中心：我的模型、测试记录、API Key 管理

#### 4.3 UI/UX 要点

- 推流区域采用深色背景，突出视频内容
- 渲染结果使用高对比度颜色，红色框、绿色关键点，如果有多个类别使用不同的颜色需要构建一个颜色类别映射表
- 提供"延迟指示器"（如：当前端到端延迟：320ms）
- 错误提示友好（如推流断开、模型不兼容等）

### 5. 技术栈建议

| 模块 | 技术选型建议 |
| --- | --- |
| 前端 | React + TypeScript + WebRTC (for camera) + Canvas/WebGL (渲染) + Ant Design |
| 后端 | Python (FastAPI)  |
| 模型推理引擎 | Triton Inference Server（需要实现pytorch,onnx,tensorrt模型的推理） |
| 流媒体处理 | FFmpeg + SRS（Simple Realtime Server）|
| 数据库 | PostgreSQL（结构化元数据） + Redis（会话/缓存） |
| 部署 | Docker（+Kubernetes（v2阶段实现）） |
| CDN/存储 | 对象存储（MinIO（开源自建）） |

> 注：为降低延迟，建议将推理服务与流媒体服务部署在同一可用区。

### 6. 非功能性需求

#### 6.1 性能

- 端到端延迟：从推流输入到前端渲染 ≤ 500ms（局域网环境），公网 ≤ 1.5s
- 并发能力：MVP 支持 ≥ 50 路并发推流测试
- 模型加载时间：热门模型预加载，冷启动 ≤ 5s

#### 6.2 安全

- 所有 API 调用需 API Key 认证
- 用户模型隔离（私有模型不可被他人访问）
- 推流地址带时效 Token（防止盗链）
- 输入内容安全扫描（防恶意文件上传）
- HTTPS 全站加密

#### 6.3 可靠性

- 推流中断自动重连（前端提示）
- 推理服务异常自动降级（返回错误码而非崩溃）
- 日志监控（Prometheus + Grafana）

#### 6.4 可扩展性

- 模型推理模块插件化，便于新增框架支持
- 流协议可扩展（未来支持 SRT、NDI 等）

#### 6.5 合规性

- 用户数据隐私保护（符合 GDPR/CCPA）
- 模型版权归属明确（上传者保留所有权）
